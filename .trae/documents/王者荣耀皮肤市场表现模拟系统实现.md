## 二、配置文件设计

- **文件路径**: `src/config/skin-simulate.config.ts`
- **功能**: 集中管理所有模拟参数，方便用户修改
- **包含内容**:
  - **基础配置**:
    - 版本信息
    - 调试开关
  - **英雄配置**:
    - 英雄强度等级定义（T0-T5）
    - 各等级对应的出场率范围
    - 各等级对应的用户基数范围
  - **皮肤配置**:
    - 不同品质皮肤的基础参数（伴生皮、勇者皮、史诗皮、传说皮、限定皮）
    - 设计契合度默认值

  - **市场配置**:
    - 基础销量、讨论量、付费率
    - 价格体系（不同品质皮肤的基准价格）
    - 获取难度映射表

  - **权重配置**:
    - 静态基础权重（英雄基础盘、皮肤核心吸引力、获取决策门槛、外部舆情热度）

    - 视觉稀缺感映射公式参数
    - 获取难度映射公式参数
    - 负面舆情映射公式参数
  - **随机事件配置**:
    - 事件频率（每两周）
    - 每个职业推荐英雄数量（1个）
    - 总推荐英雄数量（6个）
    - 主播推荐效果系数
  - **UI配置**:
    - 颜色映射
    - 显示格式

## 五、随机事件系统详细设计

### 1. 英雄强度等级定义
| 等级 | 名称 | 出场率范围 | 用户基数范围 | 配置项 |
|------|------|------------|--------------|--------|
| T0 | 版本之子 | 0.8-0.9 | 0.7-0.9 | `hero.strengthLevels.T0` |
| T1 | 强势英雄 | 0.6-0.8 | 0.6-0.8 | `hero.strengthLevels.T1` |
| T2 | 平衡英雄 | 0.4-0.6 | 0.5-0.7 | `hero.strengthLevels.T2` |
| T3 | 弱势英雄 | 0.3-0.4 | 0.3-0.5 | `hero.strengthLevels.T3` |
| T4 | 冷门英雄 | 0.2-0.3 | 0.2-0.3 | `hero.strengthLevels.T4` |
| T5 | 极冷门英雄 | 0.1-0.2 | 0.1-0.2 | `hero.strengthLevels.T5` |

### 2. 随机事件生成逻辑
```typescript
// 从配置文件读取参数
const { frequency, professions, heroesPerProfession } = SIMULATE_CONFIG.randomEvent;

// 每两周触发一次事件
if (currentDate - lastEventDate >= frequency) {
  // 为每个职业推荐指定数量的英雄
  for (const profession of professions) {
    // 随机选择该职业的英雄
    const professionHeroes = getHeroesByProfession(profession);
    const recommendedHeroes = shuffle(professionHeroes).slice(0, heroesPerProfession);
    // 随机分配强度等级
    recommendedHeroes.forEach(hero => {
      hero.strengthLevel = getRandomStrengthLevel();
      hero.popularity = calculatePopularity(hero.strengthLevel);
    });
  }
  // 保存事件记录
  saveRandomEvent();
}
```

## 七、使用说明

1. **修改模拟参数**:
   - 打开 `src/config/skin-simulate.config.ts`
   - 修改对应的配置项
   - 保存后立即生效，无需重新编译

2. **调整英雄强度等级**:
   - 在 `hero.strengthLevels` 配置中修改各等级的出场率和用户基数范围
   - 系统会自动应用新的强度等级定义

3. **修改随机事件频率**:
   - 在 `randomEvent.frequency` 配置中修改事件间隔（单位：毫秒）
   - 系统会按照新的频率生成随机事件

4. **调整权重配置**:
   - 在 `weights.baseWeights` 配置中修改各维度的权重
   - 系统会使用新的权重进行市场表现计算

通过以上实现，将完整落地文档中的加权模型，并实现每两周推荐六个职业强度英雄的随机事件系统。所有参数都集中在配置文件中，用户可以方便地修改各种模拟参数，而无需修改代码。
