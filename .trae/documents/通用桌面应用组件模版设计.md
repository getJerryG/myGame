# 通用桌面应用组件模版设计

## 1. 设计目标

* 统一管理所有桌面应用的窗口参数

* 统一所有桌面应用的视觉风格

* 实现统一的视口尺寸适配

* 提供通用的窗口控制功能

* 支持模块化的内容展示

## 2. 组件结构设计

### 2.1 主容器组件：`DesktopAppContainer`

* **功能**：作为所有桌面应用的统一容器，管理窗口参数和样式

* **位置**：`src/components/DesktopAppContainer.vue`

* **核心功能**：

  * 统一的窗口标题栏和控制按钮

  * 通用的拖拽功能

  * 窗口状态管理（最大化、最小化、关闭）

  * 统一的样式管理

  * 视口尺寸适配

### 2.2 内容区域组件

* **功能**：各个具体应用的内容展示

* **位置**：`src/components/apps/` 目录下的各个应用组件

* **设计要求**：所有应用组件遵循统一的接口规范

## 3. 统一窗口参数管理

### 3.1 窗口状态接口

```typescript
interface WindowState {
  position: { x: number; y: number };
  size: { width: number; height: number };
  isMaximized: boolean;
  isMinimized: boolean;
  activeModule: string;
}
```

### 3.2 窗口参数默认值

```typescript
const DEFAULT_WINDOW_SIZE = { width: 800, height: 600 };
const MIN_WINDOW_SIZE = { width: 600, height: 400 };
const MAX_WINDOW_SIZE = { 
  width: window.innerWidth, 
  height: window.innerHeight - 48 // 减去任务栏高度
};
```

## 6. 组件接口设计

### 6.1 DesktopAppContainer Props

```typescript
interface DesktopAppContainerProps {
  app: {
    id: string;
    name: string;
    icon: string;
    modules: Array<{
      id: string;
      name: string;
    }>;
    coreData?: Record<string, any>;
  };
  windowState: WindowState;
  gameData?: any;
}
```

### 6.2 DesktopAppContainer Emits

```typescript
type DesktopAppContainerEmits = {
  close: [];
  minimize: [];
  maximize: [];
  update: [windowState: WindowState];
  'app-installed'?: [app: any];
};
```

## 7. 实现方案

### 7.1 组件实现步骤

1. 创建 `DesktopAppContainer.vue` 组件，实现统一的窗口容器
2. 提取现有的 `DesktopModal.vue` 中的通用功能到新组件
3. 设计统一的样式主题
4. 实现视口尺寸适配功能
5. 更新现有的桌面应用，使用新的通用组件

### 7.2 样式实现方式

* 使用 CSS 变量定义主题样式

* 提供响应式布局工具类

* 实现统一的组件样式

### 7.3 视口适配实现

* 监听窗口大小变化事件

* 实现窗口大小调整时的内容适配

* 提供响应式布局示例

## 8. 迁移计划

1. 创建 `DesktopAppContainer` 组件
2. 更新 `DesktopSystem.vue`，使用新的容器组件
3. 确保所有桌面应用兼容新的组件接口
4. 测试所有桌面应用的功能和样式
5. 逐步替换旧的 `DesktopModal` 组件

## 9. 预期效果

* 所有桌面应用具有统一的视觉风格

* 统一的窗口控制和拖拽体验

* 良好的视口尺寸适配

* 提高代码复用性和可维护性

* 简化新应用的开发流程
