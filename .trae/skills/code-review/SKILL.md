# 代码审查技能

## 功能介绍

该技能用于对代码进行全面的审查和评估，帮助提高代码质量和开发效率。

### 核心功能

1. **代码质量审查**
   - 检查代码结构和组织
   - 评估代码可读性
   - 检查命名规范
   - 评估代码复杂度

2. **最佳实践检查**
   - 检查语言特定的最佳实践
   - 评估设计模式的使用
   - 检查代码复用性
   - 评估注释质量

3. **潜在 Bug 识别**
   - 检查常见的编程错误
   - 评估错误处理机制
   - 检查边界条件处理
   - 识别潜在的安全漏洞

4. **代码优化建议**
   - 性能优化建议
   - 内存使用优化
   - 算法效率改进
   - 代码简化建议

## 使用指南

### 何时调用

- 当用户要求进行代码审查时
- 当代码提交前需要审查时
- 当需要检查代码质量时
- 当需要优化现有代码时

### 调用方式

直接使用 `code-review` 技能，后跟需要审查的代码或文件路径。

## 示例

### 示例 1：审查单个文件

```
code-review ./src/components/Button.tsx
```

### 示例 2：审查代码片段

```
code-review <<EOF
function calculateTotal(prices) {
  let total = 0;
  for (let i = 0; i < prices.length; i++) {
    total += prices[i];
  }
  return total;
}
EOF
```

### 示例 3：审查整个目录

```
code-review ./src/utils/
```

## 审查标准

该技能遵循以下审查标准：

1. **可读性**：代码是否易于理解和维护
2. **可维护性**：代码是否易于修改和扩展
3. **可靠性**：代码是否稳定，不易出错
4. **性能**：代码是否高效运行
5. **安全性**：代码是否存在安全隐患
6. **可测试性**：代码是否易于测试

## 输出格式

审查结果将以结构化方式输出，包括：

- 总体评分
- 主要问题和建议
- 具体代码位置和改进建议
- 最佳实践遵循情况
- 潜在风险警告

## 支持的语言

该技能支持多种编程语言，包括但不限于：

- JavaScript/TypeScript
- Python
- Java
- C#
- Go
- Ruby
- PHP

## 注意事项

1. 代码审查结果仅供参考，最终决策权在开发团队手中
2. 审查过程可能需要一定时间，具体取决于代码规模
3. 对于大型代码库，建议分模块进行审查
4. 审查结果应结合项目实际情况进行评估和采纳
