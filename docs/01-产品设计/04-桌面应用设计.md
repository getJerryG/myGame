# 桌面应用设计详细说明

> **版本：V1.0 | 日期：2026-02-27**
> **适用范围：游戏策划模拟经营游戏**
> **父文档：游戏策划模拟经营游戏PRD.md**

---

## 1. 独立拆分版应用设计

适合中级 - 策划总监阶段，英雄、皮肤、活动、发布都是策划日常高频操作的核心模块，独立成桌面应用可一键直达，无需进入内部分页，提升操作效率，完全保留原有的 6 个应用，仅做职责边界的清晰划分，避免功能重叠：

### 1.1 原有应用明确核心职责（无重叠）


|应用名称|核心职责|
|--------|--------|
|钱包|唯一财务入口：资金余额、每日收支流水、研发 / 联动 / 活动扣费记录、流水统计|
|聊天|唯一沟通入口：与研发 / 美术 / 运营 / 老板的工作沟通、玩家舆情反馈、随机事件弹窗处理、联动合作洽谈|
|英雄|唯一英雄全生命周期管理：英雄立项、研发进度查看、体验服数值调整、英雄胜率 / Ban 率数据监控|
|皮肤|唯一皮肤全生命周期管理：皮肤立项、风格 / 品质选择、研发进度查看、联动 IP 绑定、销量数据监控|
|活动|唯一运营活动管理：活动类型 / 力度 / 周期选择、活动效果监控、历史活动记录|
|游戏发布|唯一版本上线入口：体验服 / 正式服切换、英雄 / 皮肤 / 活动的上线审批、版本更新公告发布、炸服 / BUG 修复处理|

## 2. 必须补充的应用清单

完全匹配之前的 PRD 玩法模块，和现有应用无功能重叠，同时严格遵循「应用下载安装 + 职级解锁」的核心设定，分为 开局必装系统应用 和 职级解锁高级应用 两类。

### 2.1 开局必装系统应用（见习策划开局就有，桌面默认存在）


|应用名称|核心作用|核心内容|不可替代原因|
|--------|--------|--------|----------|
|应用商店|完美落地「所有游戏功能以应用的形式下载和使用」的设定，是所有高级应用的唯一获取入口|展示全量可解锁应用，标注每个应用的解锁职级要求，达到条件即可点击「下载安装」，安装后自动出现在电脑桌面|把「职级解锁功能」转化为了更有沉浸感的「下载安装应用」，完全贴合电脑桌面模拟的核心设定|
|数据中心|对应 PRD 里的「核心数据面板 + 每日结算报表」，是玩家所有决策的数据依据|每日结算报表、游戏总流水 / 热度 / 口碑实时数据、英雄胜率 / 登场率 / Ban 率、皮肤销量明细、下载量 / 活跃度数据，全可视化图表展示|玩家每天结算后必须第一时间看数据，是所有操作的前提，无法整合到其他应用，必须独立直达|
|策划成长|对应 PRD 里的「职级体系 + 升级规则」，玩家成长的核心锚点|当前职级 / 小等级、升级进度条、下一级升级条件、已解锁 / 未解锁权限、历史升级奖励记录|玩家随时需要查看升级进度，调整自己的运营策略，是核心成长模块，必须独立|
|系统设置|电脑桌面的基础系统功能，保障游戏基础运行|游戏存档 / 读档、音量设置、画面设置、游戏退出、新手引导重置|系统级基础功能，必须独立存在|

### 2.2 职级解锁高级应用


|应用名称|解锁职级|核心职责|对应 PRD 模块|
|--------|--------|--------|------------|
|任务中心|初级策划 III 级|日常任务、主线任务、成就任务的查看、奖励领取，给玩家明确的短期成长目标|目标任务模块|
|联动中心|中级策划 V 级|全量可联动 IP / 工作室展示、联动项目管理、联动进度查看、联动结果复盘，和「聊天」里的联动沟通做区分（聊天是沟通过程，这里是项目管理）|联动合作系统|
|舆情中心|高级策划 V 级|玩家社区评论、口碑变化明细、玩家满意度统计、负面舆情预警，和「聊天」里的舆情提醒做区分（聊天是实时提醒，这里是全量数据管理）|市场情绪 / 玩家反馈模块|
|事件日志|中级策划 III 级|历史随机事件全记录、处理方案、对游戏数据的影响复盘，方便玩家调整后续决策|随机事件系统|
|渠道投放|资深策划 V 级|市场推广渠道选择、投放力度设置、下载量提升效果监控，贴合 MOBA 游戏的市场运营逻辑|市场推广 / 下载量模块|

## 3. 全应用详细交互设计

全局交互前提：只有在「玩家操作阶段」（每日 24 点结算完成后），才能打开应用进行操作；玩家点击「结束今日 → 进入下一天」后，所有应用自动锁定，不可操作，直至次日结算完成，完全贴合时间系统规则。

所有应用均模拟 Windows 桌面应用交互：支持窗口化打开、最小化、最大化、关闭，可同时打开多个应用窗口（比如一边看数据中心，一边调整英雄数值）。

### 3.1 开局必装系统应用

#### 3.1.1 应用商店

**界面结构：**

- 顶部：应用分类标签（全部、系统应用、研发类、运营类、数据类）
- 主体：应用卡片网格，每张卡片包含：应用图标、应用名称、解锁职级、功能简介、「下载安装」/「已安装」按钮

**核心交互流程：**

- 玩家点击桌面「应用商店」图标，打开窗口
- 玩家浏览应用列表，灰色不可点击的卡片为「未达到解锁职级」，高亮卡片为「可安装」
- 玩家点击「下载安装」按钮，弹出确认弹窗（仅 2 个选项：「确认安装」/「取消」）
- 点击确认后，自动完成安装，应用图标出现在桌面，弹窗提示「安装成功」

**无表单规则：** 全程仅点击按钮选择，无任何输入框

#### 3.1.2 数据中心

**界面结构：**

- 顶部：数据分类标签（今日报表、运营数据、英雄数据、皮肤数据）
- 主体：可视化数据卡片 + 折线图 / 柱状图，无任何可输入内容
- 底部：「刷新数据」按钮

**核心交互流程：**

- 玩家点击桌面「数据中心」图标，打开窗口，默认展示「今日报表」
- 玩家点击顶部标签，切换不同维度的数据
- 点击「刷新数据」，实时更新最新数值
- 点击单张数据卡片，可放大查看详细明细（仅查看，无操作）

**无表单规则：** 纯数据查看，无任何可输入内容，仅点击切换标签 / 刷新

#### 3.1.3 策划成长

**界面结构：**

- 顶部：当前职级展示（如「见习策划 III 级」）、升级经验进度条
- 中部：2 个标签页「升级条件」「已解锁权限」
  - 「升级条件」页：下一级需要达成的所有条件，已完成的标绿，未完成的标灰
  - 「已解锁权限」页：当前职级可使用的所有功能 / 应用列表
- 底部：「历史奖励记录」按钮

**核心交互流程：**

- 玩家点击桌面「策划成长」图标，打开窗口
- 玩家切换标签页，查看升级条件和已解锁权限
- 达到升级条件时，窗口自动弹出晋升弹窗，仅 1 个「确认晋升」按钮
- 点击确认后，自动完成职级提升，发放奖励，更新界面信息

**无表单规则：** 纯信息查看 + 晋升确认按钮，无任何输入内容

#### 3.1.4 系统设置

**界面结构：**

- 顶部：设置分类标签（存档管理、画面设置、音效设置、新手引导）
- 主体：选项式设置项，每个设置项包含 2-3 个单选按钮
- 底部：「保存设置」按钮

**核心交互流程：**

- 玩家点击桌面「系统设置」图标，打开窗口
- 玩家切换标签页，选择对应设置项的选项
- 点击「保存设置」，弹窗提示「设置保存成功」

**无表单规则：** 全程仅点击单选按钮，无任何输入框

### 3.2 原有核心应用

#### 3.2.1 钱包

**界面结构：**

- 顶部：当前资金余额大字展示、累计总流水
- 中部：2 个标签页「收支明细」「资产统计」
  - 「收支明细」页：每日收入 / 扣费记录、金额、收支原因
  - 「资产统计」页：月度流水折线图、收入来源占比饼图

**核心交互流程：**

- 玩家点击桌面「钱包」图标，打开窗口，默认展示余额和今日收支
- 玩家切换标签页，查看明细和统计数据
- 所有研发 / 活动 / 联动的扣费，自动同步到钱包明细，弹窗提示扣费金额和原因

**无表单规则：** 纯数据查看，无任何输入 / 操作内容

#### 3.2.2 聊天

**界面结构：**

- 左侧：聊天对象列表（研发部、美术部、运营部、玩家社区、老板、IP 合作方）
- 右侧：聊天窗口，系统自动生成对话内容，底部是选项式回复按钮

**核心交互流程：**

- 玩家点击桌面「聊天」图标，打开窗口
- 随机事件触发时，对应聊天对象会出现红色未读提示，玩家点击进入聊天窗口
- 系统自动生成事件内容和对话，底部给出 2-3 个处理方案选项按钮（如「紧急修复 BUG」「发布补偿公告」「暂时观望」）
- 玩家点击对应选项按钮，系统自动生成回复内容，同步返回事件处理结果，更新游戏数据
- 联动洽谈时，系统自动生成合作方对话，底部给出「同意合作」「拒绝合作」「进一步沟通」选项

**无表单规则：** 全程仅点击选项按钮回复，无任何输入框，所有对话内容由系统自动生成

#### 3.2.3 英雄

**界面结构：**

- 顶部：2 个标签页「新建英雄」「英雄管理」
  - 「新建英雄」页：3 组选项按钮，无任何输入框
    - 职业选择：单选按钮（射手 / 法师 / 刺客 / 坦克 / 辅助）
    - 流派选择：单选按钮（站撸大核 / POKE 消耗 / 攻速走 A / 功能型 / 开团型）
    - 难度选择：单选按钮（简单 / 中等 / 高操作）
    - 底部：「确认立项」按钮
  - 「英雄管理」页：已立项英雄列表，点击单个英雄可进入详情页，查看研发进度、体验服调整选项、数据详情

**核心交互流程：**

- 玩家点击桌面「英雄」图标，打开窗口
- 新建英雄：玩家依次选择职业、流派、难度，点击「确认立项」，系统自动生成完整英雄（名称、技能、数值、背景），同步开启研发倒计时，无需玩家写任何内容
- 英雄管理：玩家点击已完成研发的英雄，进入详情页，底部给出 3 个调整选项（小幅加强 / 保持不变 / 小幅削弱），点击选项后系统自动更新英雄数值和胜率
- 研发进度每日自动推进，玩家仅可查看，不可干预

**无表单规则：** 全程仅点击单选按钮 / 确认按钮，无任何输入框，所有英雄内容由系统自动生成

#### 3.2.4 皮肤

**界面结构：**

- 顶部：2 个标签页「新建皮肤」「皮肤管理」
  - 「新建皮肤」页：4 组选项按钮
    - 绑定英雄：单选按钮（已上线英雄列表）
    - 皮肤品质：单选按钮（伴生 / 勇者 / 史诗 / 传说 / 限定 / 荣耀典藏）
    - 皮肤风格：单选按钮（古风 / 未来科技 / 机甲 / 二次元 / 运动 / 暗黑）
    - 研发方式：单选按钮（自研 / IP 联动）
    - 底部：「确认立项」按钮
  - 「皮肤管理」页：已立项皮肤列表，点击单个皮肤可查看研发进度、销量数据、特效预览

**核心交互流程：**

- 玩家点击桌面「皮肤」图标，打开窗口
- 新建皮肤：玩家依次选择绑定英雄、品质、风格、研发方式，点击「确认立项」，系统自动生成皮肤名称、特效、定价、研发周期，同步开启研发倒计时
- 皮肤管理：玩家可查看已上线皮肤的每日销量数据，无额外操作

**无表单规则：** 全程仅点击单选按钮，无任何输入框，所有皮肤内容由系统自动生成

#### 3.2.5 活动

**界面结构：**

- 顶部：2 个标签页「新建活动」「活动管理」
  - 「新建活动」页：3 组选项按钮
    - 活动类型：单选按钮（新英雄折扣 / 英雄 + 皮肤礼包 / 排位加成 / 充值返利 / 限定皮肤返场 / 联动主题活动 / 炸服补偿）
    - 活动力度：单选按钮（轻度 / 中度 / 重磅）
    - 活动周期：系统根据力度自动匹配天数，不可修改
    - 底部：「确认开启活动」按钮
  - 「活动管理」页：进行中 / 已结束活动列表，点击可查看活动效果数据

**核心交互流程：**

- 玩家点击桌面「活动」图标，打开窗口
- 新建活动：玩家依次选择活动类型、力度，点击「确认开启」，系统自动生成活动规则、奖励、周期，活动同步上线，每日自动结算效果
- 活动管理：玩家可查看活动的流水 / 热度 / 口碑变化，无额外操作

**无表单规则：** 全程仅点击单选按钮，无任何输入框，所有活动内容由系统自动生成

#### 3.2.6 游戏发布

**界面结构：**

- 顶部：2 个标签页「待发布内容」「版本历史」
  - 「待发布内容」页：已完成研发的英雄 / 皮肤 / 活动列表，每个内容右侧有 2 个单选按钮（体验服发布 / 正式服发布）
    - 底部：「确认发布」按钮
  - 「版本历史」页：过往版本更新记录、上线内容、玩家反馈

**核心交互流程：**

- 玩家点击桌面「游戏发布」图标，打开窗口
- 玩家为待发布内容选择发布渠道（体验服 / 正式服），点击「确认发布」，系统自动生成版本公告，内容同步上线，对应数据开始更新
- 出现版本 BUG / 炸服时，窗口弹出「紧急修复」按钮，点击后系统自动处理，同步生成补偿公告

**无表单规则：** 全程仅点击单选按钮 / 确认按钮，无任何输入框，所有公告内容由系统自动生成

### 3.3 高级解锁应用

#### 3.3.1 任务中心

**界面结构：**

- 顶部：任务分类标签（日常任务、主线任务、成就任务）
- 主体：任务列表，每个任务包含：任务名称、完成条件、奖励、状态（未完成 / 已完成 / 已领取）
- 底部：「一键领取所有奖励」按钮

**核心交互流程：**

- 玩家点击桌面「任务中心」图标，打开窗口
- 玩家切换标签页，查看不同类型的任务
- 已完成的任务右侧显示「领取奖励」按钮，点击后自动发放奖励
- 点击「一键领取所有奖励」，自动领取所有已完成任务的奖励

**无表单规则：** 全程仅点击按钮，无任何输入框，所有任务内容由系统自动生成

#### 3.3.2 联动中心

**界面结构：**

- 顶部：联动分类标签（全部联动、进行中、已结束）
- 主体：联动项目列表，每个项目包含：IP 名称、联动类型、进度、状态
- 底部：「开启新联动」按钮

**核心交互流程：**

- 玩家点击桌面「联动中心」图标，打开窗口
- 玩家点击「开启新联动」按钮，弹出可联动 IP 列表，每个 IP 右侧有「开启洽谈」按钮
- 点击「开启洽谈」后，自动跳转到「聊天」应用进行沟通，洽谈完成后，联动项目自动同步到联动中心
- 玩家可查看进行中联动的进度，已结束联动的结果

**无表单规则：** 全程仅点击按钮，无任何输入内容，所有联动内容由系统自动生成

#### 3.3.3 舆情中心

**界面结构：**

- 顶部：舆情分类标签（实时舆情、口碑变化、玩家满意度）
- 主体：可视化数据图表 + 舆情明细列表
- 底部：「导出舆情报告」按钮

**核心交互流程：**

- 玩家点击桌面「舆情中心」图标，打开窗口
- 玩家切换标签页，查看不同维度的舆情数据
- 点击「导出舆情报告」，系统自动生成报告，弹窗提示「报告导出成功」

**无表单规则：** 纯数据查看，无任何输入 / 操作内容

#### 3.3.4 事件日志

**界面结构：**

- 顶部：事件分类标签（全部事件、技术事件、运营事件、市场事件）
- 主体：事件列表，每个事件包含：发生时间、事件类型、事件内容、处理方案、影响结果
- 底部：「筛选事件」按钮（弹出筛选选项）

**核心交互流程：**

- 玩家点击桌面「事件日志」图标，打开窗口
- 玩家切换标签页，查看不同类型的事件
- 点击「筛选事件」，弹出筛选选项（如时间范围、事件类型），点击选择后自动筛选列表
- 点击单个事件，可查看详细的事件内容和处理结果

**无表单规则：** 纯数据查看 + 筛选选项点击，无任何输入框

#### 3.3.5 渠道投放

**界面结构：**

- 顶部：投放渠道标签（全部渠道、线上渠道、线下渠道、合作渠道）
- 主体：渠道列表，每个渠道包含：渠道名称、投放效果、投放成本、「开启投放」/「已投放」按钮
- 底部：「查看投放报告」按钮

**核心交互流程：**

- 玩家点击桌面「渠道投放」图标，打开窗口
- 玩家切换标签页，查看不同类型的投放渠道
- 玩家点击「开启投放」按钮，弹出投放力度选项（轻度 / 中度 / 重磅），点击选择后系统自动开始投放
- 点击「查看投放报告」，查看投放效果数据

**无表单规则：** 全程仅点击按钮，无任何输入框，所有投放内容由系统自动生成

## 4. 桌面全局交互规则

- **桌面默认布局**：顶部是系统状态栏（显示当前游戏时间：第 X 年 X 月 X 日、当前职级、资金余额），主体是应用图标网格，底部是任务栏（显示已打开的应用窗口，可点击切换 / 最小化）
- **应用图标规则**：未解锁的应用不会出现在桌面，仅在应用商店里可见；安装后的应用可自由拖动图标位置，模拟真实桌面自定义
- **弹窗规则**：所有确认 / 提示弹窗，最多不超过 3 个选项按钮，无任何输入框，玩家仅需点击选择
- **新手引导**：开局首次打开每个应用，会弹出极简引导提示，标注核心功能按钮，点击「我知道了」即可关闭
- **操作优先级**：随机事件弹窗 > 应用窗口操作，出现紧急事件时，会优先弹出聊天窗口提示，玩家处理完成后，才能继续其他操作
