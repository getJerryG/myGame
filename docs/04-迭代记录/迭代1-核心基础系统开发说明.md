## 2. 开发内容

### 2.1 时间系统开发

#### 2.1.1 功能实现
- 实现了基于PRD的每日四阶段流程：
  1. **新一天00:00开始**：所有进行中任务自动推进
  2. **自动运行至24:00**：系统自动结算当日所有数据
  3. **玩家操作阶段**：玩家可进行各种操作，期间时间静止
  4. **玩家点击结束今日**：时间+1天，进入次日00:00

#### 2.1.2 技术实现
- 更新了`simulationGameStateStore`，添加了以下核心方法：
  - `startNewDay()`：开始新的一天
  - `runAutoPhase()`：自动运行阶段
  - `enterPlayerTurn()`：进入玩家决策阶段
  - `endToday()`：结束今日，进入下一天
- 新增了状态属性：
  - `isPlayerTurn`：标记是否为玩家决策阶段
  - `dayPhase`：记录每日当前阶段

### 2.2 全局核心数值系统

#### 2.2.1 功能实现
- 实现了PRD中定义的全局核心数值：
  - `money`：当前资金（初始10000）
  - `reputation`：声望（初始0，范围0~5000）
  - `popularity`：游戏热度（初始50，范围0~200）
  - `wordOfMouth`：口碑（初始100，范围-100~100）
  - `totalMoney`：累计总流水（初始0）
  - `heroCount`：已上线英雄总数（初始0）
  - `skinCount`：已上线皮肤总数（初始0）

#### 2.2.2 技术实现
- 创建了新的状态管理文件`simulationCoreValuesStore.ts`
- 实现了核心数值的更新、增加、消耗等方法
- 添加了英雄和皮肤上线时的数值变化逻辑
- 实现了每日收入计算逻辑
