# 文档格式规范

## 表格格式要求

### 分隔线规范
- 表格标题行与内容行之间的分隔线必须使用 `---`
- 禁止使用长度不一的分隔线（如 `--------`）

### 格式示例
```markdown
<!-- prettier-ignore -->
| 列标题1 | 列标题2 | 列标题3 |
| --- | --- | --- |
| 内容1 | 内容2 | 内容3 |
```

### 注意事项
1. **必须添加 Prettier 忽略注释**：在表格前添加 `<!-- prettier-ignore -->`，确保 Prettier 不会自动调整分隔线长度
2. **列对齐**：保持列内容对齐，提高可读性
3. **内容简洁**：表格内容应简洁明了，避免过长文本
4. **边框样式**：使用标准 Markdown 表格格式，不添加额外边框样式

## 其他格式要求

### 标题层级
- 使用 `#` 到 `######` 表示标题层级
- 同一文档内标题层级应保持一致

### 列表格式
- 使用 `-` 作为无序列表标记
- 使用 `1.` 作为有序列表标记
- 列表缩进使用 2 个空格

### 代码块
- 使用 ``` 包裹代码块
- 代码块应指定语言类型（如 ```javascript）

## 工具配置

### Prettier
- 项目已配置 Prettier，使用 `pnpm prettier --write <文件路径>` 格式化文档
- 表格必须添加 `<!-- prettier-ignore -->` 注释以保持分隔线规范

### ESLint
- 项目已配置 ESLint 检查代码质量
- Markdown 文件可通过 `pnpm lint <文件路径>` 进行检查

## 执行流程

1. 创建或编辑文档时，遵循上述格式规范
2. 完成编辑后，运行 `pnpm prettier --write <文件路径>` 格式化文档
3. 运行 `pnpm lint <文件路径>` 检查文档质量
4. 提交代码前确保所有格式要求都已满足
