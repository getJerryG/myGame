## 3. 项目概述

### 3.1 项目背景

随着游戏行业的快速发展，游戏策划作为游戏开发的核心角色，其工作内容和成长路径越来越受到关注。本项目基于模拟经营（游戏策划）开发机制梳理，开发一款以「玩家成长（策划职级晋升）」为线索的模拟经营游戏，玩家将扮演游戏策划角色，完成从游戏基础设定到规模化运营的全流程。

### 3.2 产品定位

本产品为 MOBA 运营模拟 + 策划养成类游戏，玩家将以王者荣耀项目组主策划身份，运营`《MOBA》`游戏，通过全选项式操作完成英雄研发、皮肤制作、联动合作等工作，体验策划成长路径。

### 4.1 用户特征

核心用户为 `18-35` 岁的`游戏爱好者`、`准游戏从业者`及`策略类游戏玩家`，具备一定游戏经验。

## 5. 核心时间系统（全局主线）

### 5.1 时间格式

| 时间单位 | 内容 |
| --- | --- |
| 年 | 1 ~ 无限 |
| 月 | 1 ~ 12（每月固定 30 天） |
| 日 | 1 ~ 30 |
| 每天 | 游戏内最小时间单元 |

### 5.2 一天完整流程（固定四阶段）

| 阶段序号 | 阶段名称 | 阶段描述 |
| --- | --- | --- |
| 1 | 新一天 00:00 开始 | 所有进行中任务自动推进（研发、测试、活动倒计时），瞬间完成，无需等待 |
| 2 | 自动运行至 24:00 | 系统自动结算当日所有数据：流水、热度、口碑、英雄数据、皮肤销量、研发进度 |
| 3 | 玩家操作阶段（无时限） | 玩家可进行：英雄立项 / 调整、皮肤立项 / 联动、活动开启、事件处理、查看数据<br>期间时间静止 |
| 4 | 玩家点击【结束今日 → 进入下一天】 | 时间 + 1 天，进入次日 00:00，循环 |

### 5.3 唯一时间触发规则

- 时间前进只有一种方式：玩家主动点击「下一天」
- 24 小时过程全自动、瞬间结算
- 玩家仅在每日结算后进行决策

## 7. 策划等级体系

> **详细等级体系说明请参阅：**
> 📄 [01-产品核心备忘/02-策划等级体系详细说明.md](01-产品核心备忘/02-策划等级体系详细说明.md)

### 7.1 等级体系概述

策划等级共 8 级，每个等级又分多个子等级，共 48 个等级。

- **等级结构**：见习策划、初级策划、中级策划、高级策划、资深策划、专家策划、策划经理、策划总监
- **升级条件**：随职级提升逐步增加要求，包括口碑、英雄数量、皮肤数量、累计流水、联动次数、爆款英雄数量等
- **解锁功能**：不同等级解锁不同的功能权限，包括可立项内容、可研发皮肤类型、可开启活动、可解锁联动类型等
- **升级奖励**：每次升级可获得资金和声望奖励，奖励随等级提升而增加

## 12. 玩法功能详情

### 12.1 游戏基础设定模块

#### 12.1.1 功能描述

玩家在游戏开始时，需要完成游戏的基础设定，包括选择游戏类型、设定游戏名称等，同时可进行英雄立项的初始选择。

#### 12.1.2 交互流程

1. 玩家进入游戏，点击「开始游戏」按钮
2. 进入游戏基础设定界面，选择游戏类型（固定为 MOBA）
3. 设定游戏名称（支持自定义）
4. 点击「确定」按钮，完成游戏基础设定
5. 进入英雄初始立项选择界面，选择首个英雄的职业、流派、难度
6. 系统生成游戏核心数据面板，玩家开始游戏

#### 12.1.3 操作逻辑

- 游戏类型选择：固定为 MOBA，不可变更
- 游戏名称设定：输入框，支持中英文输入，字数限制为 2-20 个字符
- 确定按钮：点击后验证输入内容，若符合要求则进入下一步，否则提示错误信息
- 英雄初始立项：仅可选择射手职业（见习策划权限限制）

#### 12.1.4 界面设计

- 界面风格：卡通奇幻风格，色彩鲜艳，布局简洁
- 主要元素：游戏类型选择区域、游戏名称输入区域、确定按钮、英雄初始立项选择区域
- 交互反馈：选择英雄类型时，显示该类型英雄的特点和优势；输入游戏名称时，实时验证输入内容；点击确定按钮时，显示加载动画

### 12.2 策划操作模块

#### 12.2.1 功能描述

玩家通过各种策划操作，推动游戏的发展和运营，包括英雄立项 / 调整、皮肤立项 / 联动、活动开启、事件处理等。

#### 12.2.2 交互流程

1. 玩家进入游戏主界面，查看核心数据面板
2. 点击左侧导航栏的「操作」按钮，进入操作界面
3. 选择要执行的操作类型（如英雄立项、皮肤研发、开启活动）
4. 根据当前策划等级权限，选择对应操作的选项（如英雄职业、皮肤品质、活动类型）
5. 点击「执行」按钮，消耗相应资源，执行操作
6. 系统显示操作结果，更新游戏核心数据

#### 12.2.3 操作逻辑

- 操作权限：不同职级的策划拥有不同的操作权限，高级操作需要提升职级才能解锁（严格遵循策划等级体系的解锁功能）
- 资源消耗：执行操作需要消耗一定的资金和其他资源（如联动需要消耗声望和资金）
- 操作冷却：部分操作有冷却时间，需要等待一段时间才能再次执行
- 操作效果：操作效果根据操作类型、内容质量、市场环境等因素动态生成，系统自动计算结果

#### 12.2.4 界面设计

- 界面风格：卡通奇幻风格，色彩鲜艳，布局清晰
- 主要元素：操作类型导航栏、操作选项选择区域、资源消耗显示区域、执行按钮
- 交互反馈：选择操作类型时，显示该操作的说明和效果；选择操作选项时，实时计算资源消耗；执行操作时，显示加载动画和操作结果

### 12.3 目标任务模块

#### 12.3.1 功能描述

游戏设有各种目标和任务，玩家完成后可以获得丰厚的奖励，包括资金、声望、策划经验等。

#### 12.3.2 交互流程

1. 玩家进入游戏主界面，点击左侧导航栏的「目标」按钮，进入目标任务界面
2. 查看当前可用的目标和任务（包括日常任务、主线任务、成就任务）
3. 选择要完成的任务，查看任务要求和奖励
4. 根据任务要求执行相应操作
5. 任务完成后，系统自动发放奖励
6. 玩家点击「领取」按钮，获得奖励

#### 12.3.3 操作逻辑

- 任务类型：
  - 日常任务：每天刷新，完成后获得基础奖励（如资金 + 500、声望 + 10）
  - 主线任务：与策划等级晋升相关，完成后获得大量奖励和职级晋升推进
  - 成就任务：长期目标，完成后获得稀有奖励和成就称号
- 任务进度跟踪：系统实时跟踪玩家的任务进度，完成后自动提示
- 奖励发放：任务完成后，奖励自动发放到玩家背包，玩家需要手动领取

#### 12.3.4 界面设计

- 界面风格：卡通奇幻风格，色彩鲜艳，布局清晰
- 主要元素：任务类型导航栏、任务列表、任务详情区域、奖励领取按钮
- 交互反馈：任务完成时，显示提示信息；领取奖励时，显示动画效果

### 12.4 事件处理模块

#### 12.4.1 功能描述

游戏中会随机发生各种事件，如 Bug 出现、玩家反馈、市场变化等，玩家需要及时处理这些事件，否则会对游戏产生负面影响。

#### 12.4.2 交互流程

1. 游戏中随机触发事件，系统通过弹窗或通知提示玩家
2. 玩家点击事件提示，进入事件处理界面
3. 查看事件详情和影响
4. 选择处理方案（每个事件有 3 种处理方案）
5. 点击「确定」按钮，执行处理方案
6. 系统显示处理结果，更新游戏核心数据

#### 12.4.3 操作逻辑

- 事件类型：
  - 技术事件：如 Bug 出现、服务器故障等
  - 运营事件：如玩家大规模投诉、竞争对手发布新游戏等
  - 市场事件：如政策变化、市场趋势变化等
- 触发概率：普通事件 30%/ 天，重大事件 10%/ 天
- 处理方案：每个事件有 3 种处理方案，不同方案会产生不同的结果（如口碑变化、资金变化、热度变化）
- 事件影响：事件处理不当会对游戏的核心数据产生负面影响，如降低市场情绪、减少收入等

> **P0事件库（20个核心事件及数值影响）** 请参阅：
> 📄 [01-产品核心备忘/01-核心数值规则.md](01-产品核心备忘/01-核心数值规则.md) 第4章

#### 12.4.4 界面设计

- 界面风格：卡通奇幻风格，根据事件类型调整色彩和氛围（如技术事件使用蓝色系，紧急事件使用红色系）
- 主要元素：事件详情区域、处理方案选择区域、确定按钮
- 交互反馈：选择处理方案时，显示该方案的预期效果；执行处理方案时，显示加载动画和处理结果

### 12.5 核心数据面板

#### 12.5.1 功能描述

核心数据面板是玩家了解游戏运营状况的主要窗口，显示游戏的各项核心数据和指标。

#### 12.5.2 交互流程

1. 玩家进入游戏主界面，核心数据面板自动显示
2. 玩家可以通过切换标签页，查看不同维度的数据（如收入数据、用户数据、市场数据、运营数据）
3. 点击数据图表，可以查看更详细的数据信息
4. 系统定期更新数据，玩家可以点击「刷新」按钮手动刷新数据

#### 12.5.3 操作逻辑

- 数据更新频率：核心数据每天 24:00 自动更新一次，玩家可以手动刷新
- 数据维度：
  - 收入数据：显示游戏的收入情况，包括总收入、每日收入、收入趋势等
  - 用户数据：显示游戏的用户情况，包括下载量、活跃用户数、付费用户数等
  - 市场数据：显示游戏的市场表现，包括市场份额、市场排名、市场情绪等
  - 运营数据：显示游戏的运营情况，包括内容更新频率、Bug 数量、玩家满意度等
- 数据可视化：使用图表（如折线图、柱状图、饼图）直观展示数据变化趋势

#### 12.5.4 界面设计

- 界面风格：卡通奇幻风格，色彩鲜艳，布局清晰，数据可视化效果良好
- 主要元素：数据标签页、数据图表、数据指标卡片、刷新按钮
- 交互反馈：切换标签页时，平滑过渡动画；点击图表时，显示详细数据；刷新数据时，显示加载动画

## 14. 界面设计规范

### 14.1 设计风格

采用卡通奇幻风格，色彩鲜艳，饱和度高，以暖色调为主，营造轻松愉快的游戏氛围。界面元素使用圆角设计，柔和的阴影效果，卡通风格的图标和边框，增强游戏的趣味性和吸引力。

### 14.2 色彩方案

- 主色调：#FF6B6B（活泼粉）、#4ECDC4（清新绿）、#45B7D1（天空蓝）
- 辅助色：#FFD93D（阳光黄）、#6B5B95（梦幻紫）
- 文本色：#2D3748（深灰）、#FFFFFF（白色）
- 背景色：渐变背景，主色混合

### 14.3 交互效果

- 按钮悬浮时缩放 1.05 倍，添加发光效果
- 面板边框添加脉动动画
- 数据变化时有数字弹出动画
- 操作成功时有庆祝动画
- 页面切换时有平滑过渡效果

### 14.4 响应式设计

确保在不同屏幕尺寸下保持良好的游戏体验，针对移动设备优化布局和交互，如调整按钮大小、简化界面元素、优化触摸操作等。

## 15. 桌面应用设计

游戏采用模拟电脑桌面的交互方式，所有功能以应用程序的形式呈现。玩家通过下载、安装和使用不同的应用程序来进行游戏操作。

### 15.1 应用体系概述

应用体系分为三类：
1. **原有核心应用**：钱包、聊天、英雄、皮肤、活动、游戏发布
2. **开局必装系统应用**：应用商店、数据中心、策划成长、系统设置
3. **职级解锁高级应用**：任务中心、联动中心、舆情中心、事件日志、渠道投放

### 15.2 核心交互规则

- 应用支持窗口化操作（打开、最小化、最大化、关闭）
- 仅在玩家操作阶段可使用应用

### 16.2 架构设计概述

采用模块化架构，将游戏分为多个独立的模块，每个模块负责特定的功能，模块之间通过清晰的接口进行通信。

#### 16.2.1 模块划分

| 模块名称 | 功能描述 |
| --- | --- |
| 游戏核心模块 | 负责游戏的核心逻辑和数据管理，包括时间系统、数值系统、策划等级系统 |
| 界面展示模块 | 负责游戏的界面渲染和用户交互 |
| 操作管理模块 | 负责游戏中各种操作的执行和效果计算，包括英雄研发、皮肤研发、活动执行 |
| 目标任务模块 | 负责游戏目标和任务的管理和奖励发放 |
| 事件系统模块 | 负责游戏事件的生成和处理 |
| 成长系统模块 | 负责玩家的成长和职级晋升，严格遵循策划等级体系规则 |
| 数据统计模块 | 负责游戏数据的统计和分析 |

### 16.3 性能优化

1. **资源加载优化**：使用懒加载、预加载等技术，优化资源加载速度
2. **渲染优化**：使用虚拟列表、缓存等技术，优化界面渲染性能
3. **计算优化**：将复杂计算放在后台线程执行，避免阻塞主线程
4. **内存优化**：及时释放不再使用的资源，避免内存泄漏

## 17. 测试与质量保证

### 17.1 测试策略

1. **单元测试**：对核心功能和模块进行单元测试，确保代码质量
2. **集成测试**：测试模块之间的交互和集成，确保系统整体功能正常
3. **E2E 测试**：测试完整的用户流程，确保用户体验良好
4. **性能测试**：测试游戏在不同设备和网络环境下的性能表现
5. **兼容性测试**：测试游戏在不同浏览器和设备上的兼容性

### 17.2 质量标准

1. **功能完整性**：所有功能按照需求文档实现，无缺失功能
2. **功能正确性**：所有功能按照预期工作，无逻辑错误，策划等级、数值计算严格遵循文档规则
3. **性能表现**：页面加载时间≤3 秒，交互响应时间≤100ms
4. **兼容性**：支持 Chrome、Firefox、Safari 最新版本，支持桌面端和移动端
5. **稳定性**：游戏运行稳定，无崩溃、卡死等问题
6. **用户体验**：界面美观，操作流畅，反馈及时
